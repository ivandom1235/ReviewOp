# ProtoNet (V6) — Ambiguity-Aware Aspect Extraction

The ReviewOp ProtoNet system implements a few-shot, prototypical network for **Evidence-Grounded Implicit Aspect Detection**. It is designed to work with the ambiguity-aware V6 benchmarks generated by the `dataset_builder`.

## Architecture Overview
- **Hybrid Encoder**: Supports Transformer-based encoding (distilbert/deberta) with local caching for high-speed episode building.
- **Selective Inference**: Implements a 4-band decision logic for handling **Ambiguity** and **Novelty**:
  - `known`: Single or multi-label assignment based on prototype distance.
  - `boundary`: Abstention zone for low-confidence or high-ambiguity cases.
  - `novel`: Open-world discovery for out-of-distribution aspects.
- **Heavy Compute Offloading**: Sentiment classification within the joint label is offloaded to a **RunPod Flash `@Endpoint`** (`reviewop-sentiment`) with a local CPU fallback (Flan-T5).

## Core Workflow

### 1. Training
Train the prototypical matching layer on the processed V6 benchmark:
```powershell
python protonet\code\cli.py train `
    --input-type benchmark `
    --input-dir dataset_builder\output\benchmark\ambiguity_grounded `
    --epochs 50 `
    --learning-rate 0.001
```

### 2. Novelty Calibration (Optional but Recommended)
Optimize the `T_known` and `T_novel` thresholds using validation set prediction distributions:
```powershell
# First, generate validation predictions
python protonet\code\cli.py eval --split val --output protonet\output\val_preds.jsonl

# Then, calibrate thresholds
python protonet\code\calibrate_novelty.py --input protonet\output\val_preds.jsonl
```

### 3. Evaluation
Evaluate on the test set using the best checkpoint:
```powershell
python protonet\code\cli.py eval `
    --input-type benchmark `
    --input-dir dataset_builder\output\benchmark\ambiguity_grounded `
    --checkpoint protonet\output\checkpoints\best.pt `
    --split test
```

### 4. Production Export
Bundle the encoder, projection head, and calibrated prototype bank into a single production runtime artifact:
```powershell
python protonet\code\cli.py export `
    --checkpoint protonet\output\checkpoints\best.pt `
    --output protonet\metadata\model_bundle.pt
```

## Configuration
All hyperparameters (selective alpha/beta/gamma, margin, thresholds) are managed in `protonet\code\config.py` and can be overridden via `ProtonetConfig`.

## RunPod Integration
Ensure `FLASH_ENABLED=true` and `RUNPOD_API_KEY` are present in your `.env` to enable the high-compute sentiment offloading during inference.
